# 页面助手（右侧 Chat）设计

**日期：** 2026-07-14
**状态：** 已实施并验证

## 目标

在 Wiki 阅读工作区的右侧提供可折叠、可调整宽度的页面助手，帮助用户基于正在阅读的页面与多个手动附加页面进行多轮对话并修改 Wiki 内容。

页面助手必须与完整 Chat 共用会话、消息历史和 Agent 请求流。它不是独立的聊天系统。

## 已确认的产品决策

- 右侧采用常驻完整侧栏；折叠后保留窄按钮，展开后可拖拽调整宽度。
- 默认自动把当前打开的 `wiki/**/*.md` 页面作为上下文。
- 用户可手动附加多个 Wiki 页面；切换当前阅读页面时，自动页替换，手动页保留。
- 页面助手与完整 Chat 共享活动会话：双方切换或新建会话都会同步。
- 页面助手提供紧凑会话选择器和“新建会话”入口。
- 手动附加页面与写入模式均为会话级配置，并使用既有会话持久化恢复。
- 覆盖已有 Wiki 页面的默认行为是每次写入前确认；用户可针对当前会话改为直接写入。

## UI 与布局

### Wiki 工作区

`AppLayout` 在 Wiki 视图右侧显示页面助手，并复用现有右侧面板的拖拽与宽度限制：最小约 250px，最大工作区宽度的 50%。

页面助手与 Research 面板共用右侧区域、互斥展示。关闭页面助手后恢复 Research 面板的既有行为。Chat、设置、来源、图谱等其他主视图不改变。

### 页面助手结构

从上到下：

1. 标题栏：展开/折叠、关闭、跳转到完整 Chat。
2. 会话选择器：显示活动会话、切换既有会话、创建会话；流式生成期间禁用。
3. 上下文页签：显示不可移除的当前自动页面；显示可移除的手动 Wiki 页面，并提供追加入口。
4. 会话写入模式：`confirm`（默认）或 `direct`；流式生成期间禁用。
5. 可复用 Chat 主体：消息流、Agent 活动、输入框及已有的流式/取消体验；不渲染会话列表。

完整 Chat 仍保留会话列表与现有会话管理功能。

## 状态和持久化

扩展 `Conversation`：

- `manualContextFiles?: string[]`：用户手动附加的、去重后的项目相对 `wiki/**/*.md` 路径。
- `wikiWriteMode?: "confirm" | "direct"`：会话写入策略；缺失时视为 `confirm`。

自动上下文不持久化。它从 Wiki 预览当前的 `selectedFile` 派生，只有该文件位于项目 `wiki/` 下且为 Markdown 时才有效。

发往 Agent 的 `contextFiles` 按下列顺序构建，并做去重和有效性校验：

1. 当前自动页面（如存在）；
2. 当前会话的手动附加页面。

切换会话只切换手动页和写入模式；自动页继续跟随当前阅读页面。旧的会话 JSON 没有新增字段时必须正常加载，并采用空手动页和 `confirm` 默认值。

## 写入确认协议

### 安全边界

Rust Agent Runtime 必须强制执行写入确认，不能仅由前端决定。原有 `wiki.write_page` 的目录限制、Markdown 限制、隐藏路径拒绝、权限检查及文件历史记录继续生效。

### Confirm 模式

当模型尝试覆盖已存在 Wiki 页面时：

1. 运行时不执行写入，生成与会话和本次请求绑定的待确认动作。
2. 返回结构化数据：动作标识、页面路径、是否已存在、候选内容和变更摘要。
3. 前端渲染确认卡，用户可确认或取消。
4. 确认请求必须带回该动作标识；后端重新校验会话、动作有效期、路径和内容后，才以 `allowOverwrite=true` 调用既有安全写入路径。
5. 取消或过期的动作无法执行。

### Direct 模式

当当前会话选择 `direct` 时，只有用户明确要求创建或更新 Wiki 内容时，才允许 Agent 使用现有覆盖写入路径。写入仍记录文件历史和 Agent 文件变更。

新增页面保留当前行为，不需要覆盖确认。任何模式都不得写入 `raw/sources/`。

## 异常与交互边界

- 当前文件不属于 `wiki/` Markdown、尚未加载或被删除时，不附加自动上下文，并显示非阻塞说明。
- 失效的手动页面在发送时被排除并提示；其余有效页面正常发送。
- 折叠右栏不终止流式生成。
- 写入失败、过期确认或文件冲突必须显示具体错误；只有成功后才刷新文件树、递增数据版本并刷新当前预览。
- 用户未提交的既有工作区改动不应被本功能修改或覆盖。

## 实施结构

- 将当前大型 `ChatPanel` 拆为会话列表和可嵌入 Chat 主体，使完整 Chat 与页面助手共用消息发送、流式、Agent 事件和输入逻辑。
- 增加页面助手壳层，集中处理会话选择、页面上下文、写入模式与侧栏控制。
- 在 `AppLayout` / `ContentArea` / Wiki 预览组合页面助手，保持其他视图不变。
- 扩展 Zustand Chat store、持久化读取/写入与自动保存流程以支持会话级配置。
- 扩展 Agent API 类型、运行时确认存储和确认执行路径，避免前端绕过确认。
- 同步更新 `en.json` 和 `zh.json`。

## 验证

- Store：切换会话恢复手动上下文/写入模式；自动页替换、去重和路径筛选正确。
- 持久化：新字段保存/恢复；旧会话数据兼容默认值。
- UI：展开/折叠、宽度、会话切换/新建双向同步、手动页添加和移除、跳转完整 Chat。
- 请求：发送的上下文为自动页加有效手动页且无重复。
- 写入：`confirm` 模式确认前不覆盖、确认后覆盖并刷新、取消/失效确认拒绝；`direct` 模式满足显式用户更新请求时可写入。
- i18n：中英文键齐全，并通过现有 parity 测试。
